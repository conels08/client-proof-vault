# Post-MVP Execution Checklist

## Phase A: Value Features That Justify Payment Soon

- [ ] **Testimonial Request Links:** Add a public testimonial request URL per user/page with a submission form that stores incoming testimonials as `Pending`.
- [ ] **Submission moderation:** Build approve/reject controls so only approved testimonials become visible on the proof page.
- [ ] **Share Mode:** Implement a condensed shareable proof-page view focused on trust highlights.
- [ ] **Copy share summary:** Add a one-click copy action for a short summary snippet users can paste into outreach messages.
- [ ] **Verified Email badge:** Display badge based on authenticated email verification state.
- [ ] **Verified Domain badge:** Implement DNS TXT verification flow and display badge after successful verification.

**Definition of Done (Phase A):** A user can collect external testimonials safely, curate them before publish, share a concise trust view/snippet, and display core verification badges.

## Phase B: Make It Premium + Measurable

- [ ] **Analytics panel shell:** Add a dashboard analytics area for proof-page performance.
- [ ] **Views over time:** Show simple time-series views (e.g., daily buckets).
- [ ] **Total views metric:** Show all-time views clearly alongside trend data.
- [ ] **Public page visual polish:** Improve typography hierarchy, spacing rhythm, and section readability.
- [ ] **Image handling polish:** Improve responsive image presentation and consistency across section types.
- [ ] **Section ordering polish:** Refine ordering UI and public rendering order clarity.
- [ ] **Custom domains setup:** Add domain mapping flow for assigning a custom domain to a proof page.
- [ ] **Custom domains validation:** Add DNS/ownership checks and clear status states (pending, verified, active).

**Definition of Done (Phase B):** The product demonstrates measurable page performance and premium-grade public presentation, with working custom-domain mapping.

## Phase C: AI Assist (Low-Risk)

- [ ] **AI headline + bio optimizer:** Provide rewrite suggestions users can accept or discard.
- [ ] **AI testimonial summarizer/shortener (optional):** Offer concise variants for existing testimonial text.
- [ ] **AI metric framing suggestions (optional):** Suggest clearer phrasing for user-provided metric claims.
- [ ] **Human-in-the-loop controls:** Ensure all AI outputs require user confirmation before save/publish.
- [ ] **Safety guardrails:** Prevent fabrication by only transforming user-provided content.

**Definition of Done (Phase C):** Users can use optional AI assistance to improve wording quality while retaining full editorial control and claim integrity.
